# 📊 프로젝트 리팩토링 분석 보고서 (Analysis Report)

본 보고서는 `persona-style` 프로젝트의 현재 상태를 진단하고, 유지보수성, 확장성 및 성능 향상을 위한 리팩토링 대상을 식별합니다.

## 1. 종합 진단 결과

현재 프로젝트는 기능적으로 잘 작동하고 있으나, 코드의 조직화와 최신 Next.js/TypeScript 권장 패턴 적용 면에서 개선의 여지가 많습니다. 특히 비즈니스 로직이 API 라우트에 집중되어 있고, 프론트엔드 컴포넌트 구조가 파편화되어 있습니다.

## 2. 주요 리팩토링 대상 (Key Targets)

### 2.1. 프로젝트 구조 및 조직화 (Feature-Based Architecture)
- **현황**: `src/components/features`에 UI 컴포넌트와 일부 도메인 로직이 혼재되어 있음.
- **문제점**: 도메인 관련 로직(API 호출, 훅, 타입)을 찾기 어렵고 기능 간 결합도가 높아질 위험이 있음.
- **개선안**: `src/features/{feature-name}` 구조로 전환하여 관련 로직을 응집시킴.
  - `src/features/analyze`: 분석 관련 로직
  - `src/features/match`: 매칭 관련 로직
  - `src/features/premium`: 프리미엄 레포트 관련 로직

### 2.2. API 라우트 및 서비스 레이어 분리
- **현황**: `src/app/api/analyze/route.ts`에 AI 프롬프트 생성, 모델 재시도, 폴백 로직 등이 모두 포함됨.
- **문제점**: 라우트 핸들러가 비대해져 테스트와 유지보수가 어려움.
- **개선안**: `Backend Development Guidelines`에 따라 서비스 레이어를 도입하여 비즈니스 로직을 이관.
  - `src/features/analyze/services/analysisService.ts` 생성.

### 2.3. Next.js 서버/클라이언트 컴포넌트 최적화
- **현황**: `app/page.tsx` 등 메인 페이지가 `"use client"`로 선언되어 정적 콘텐츠까지 클라이언트 사이드에서 렌더링됨.
- **문제점**: 번들 크기 증가 및 초기 로딩 성능 저하.
- **개선안**: 페이지를 서버 컴포넌트로 유지하고, 상호작용이 필요한 부분만 클라이언트 컴포넌트로 분리.

### 2.4. 타입 안정성 강화 (Zero Any Policy)
- **현황**: `src/lib/logger.ts` 등에서 `any` 타입을 사용 중.
- **문제점**: 런타임 에러 위험 및 개발자 경험 저하.
- **개선안**: `any`를 `unknown` 또는 구체적인 인터페이스로 대체.

### 2.5. AI 연동 로직 표준화
- **현황**: `gemini.ts`의 모델 명칭 오류 가능성(`gemini-2.5-flash-image`) 및 재시도 로직의 수동 구현.
- **문제점**: AI 응답의 안정성 및 예측 가능성 저하.
- **개선안**: 유효한 모델 명칭 사용 및 공통 재시도 유틸리티 도입.

### 2.6. Supabase 클라이언트 관리 개선
- **현황**: `src/lib/supabase.ts`에서 브라우저 클라이언트만 일괄 export.
- **문제점**: 서버 사이드에서 잘못된 클라이언트를 사용할 위험이 있음.
- **개선안**: `client`와 `server` 클라이언트의 용도를 명확히 구분하는 인덱스 구조 도입.

## 3. 리팩토링 우선순위

1. **P0**: API 라우트 로직 분리 및 서비스 레이어 구축 (안정성 및 테스트 가능성 확보)
2. **P0**: 프로젝트 구조를 Feature-based로 재편 (유지보수성 확보)
3. **P1**: Next.js 서버/클라이언트 컴포넌트 최적화 (성능 향상)
4. **P1**: 타입 안정성 강화 및 `any` 제거 (코드 품질 향상)
5. **P2**: 유틸리티 및 클린 코드 정제 (가독성 향상)

## 4. 기대 효과

- **유지보수성**: 기능 단위로 코드가 응집되어 수정 및 확장이 용이해짐.
- **성능**: 불필요한 클라이언트 사이드 자바스크립트 실행을 줄여 로딩 속도 개선.
- **안정성**: 엄격한 타입 체크와 표준화된 에러 처리를 통해 런타임 에러 감소.
