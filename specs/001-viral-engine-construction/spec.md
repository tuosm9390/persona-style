## User Scenarios & Testing *(mandatory)*

### User Story 1 - 공유용 아이덴티티 카드 고도화 (Priority: P1)

사용자가 자신의 페르소나 분석 결과를 인스타그램 스토리나 틱톡 등 소셜 미디어에 즉시 공유할 수 있는 고퀄리티의 시각화 카드를 생성합니다. 단순한 텍스트 위주가 아닌, 페르소나 유형에 따른 동적 디자인(색상, 패턴, 레이아웃)이 적용된 이미지입니다.

**Why this priority**: 서비스의 핵심 바이럴 요소이며, 시각적 만족도가 사용자 유입의 가장 큰 동기 부여가 되기 때문에 최우선 순위로 설정합니다.

**Independent Test**: 사용자가 분석을 완료한 후 '공유 카드 생성' 버튼을 클릭하면, 3초 이내에 모바일 공유에 최적화된 고해상도 이미지가 생성되어 다운로드하거나 소셜 앱으로 전송할 수 있는지 확인합니다.

**Acceptance Scenarios**:

1. **Given** 페르소나 분석이 완료된 상태에서, **When** 사용자가 '공유 이미지 저장' 버튼을 누르면, **Then** 분석 결과(닉네임, 페르소나 유형, 핵심 키워드, 디자인 요소)가 포함된 PNG 이미지가 생성된다.
2. **Given** 생성된 공유 카드에서, **When** 소셜 미디어 플랫폼 규격(9:16 등)에 맞는지 확인하면, **Then** 이미지의 텍스트와 디자인 요소가 잘리지 않고 선명하게 노출된다.

---

### User Story 2 - 페르소나 매칭 (성향 궁합 테스트) (Priority: P2)

두 명의 사용자(또는 본인의 다른 데이터)의 페르소나 분석 결과를 대조하여 서로의 '궁합 점수'와 '관계 팁'을 제공합니다. 이는 친구나 연인과 함께 서비스를 즐기게 만드는 강력한 네트워크 효과를 유도합니다.

**Why this priority**: 일회성 분석을 넘어 사용자 간의 상호작용을 유도하고 새로운 사용자를 초대하는 트리거 역할을 하므로 높은 순위를 부여합니다.

**Independent Test**: 두 개의 서로 다른 분석 결과 ID를 입력하거나 링크를 공유받았을 때, 시스템이 두 데이터의 유사도와 보완점을 분석하여 '케미스트리 리포트'를 생성하는지 테스트합니다.

**Acceptance Scenarios**:

1. **Given** 두 개의 페르소나 분석 데이터가 존재할 때, **When** '궁합 확인하기' 기능을 실행하면, **Then** 0~100점 사이의 점수와 함께 두 성향의 장단점이 포함된 요약 결과가 출력된다.

---

### User Story 3 - 글로벌 페르소나 트렌드 대시보드 (Priority: P3)

전체 사용자들의 분석 데이터를 익명으로 집계하여 현재 유행하는 페르소나 트렌드를 실시간 시각화하여 보여줍니다. 사용자는 자신의 페르소나가 전체에서 어느 위치에 있는지(희귀도 등) 확인할 수 있습니다.

**Why this priority**: 서비스 내 체류 시간을 늘리고 '나의 특별함'을 확인하고 싶은 욕구를 자극하지만, 개별 사용자의 직접적인 가치보다는 커뮤니티적 성격이 강하므로 P3로 설정합니다.

**Independent Test**: 메인 피드나 트렌드 페이지 접속 시, 전체 페르소나 분포도와 실시간 인기 키워드가 차트 형태로 정상 출력되는지 확인합니다.

**Acceptance Scenarios**:

1. **Given** 다수의 사용자가 분석을 완료한 상태에서, **When** 트렌드 페이지에 접속하면, **Then** 가장 많이 도출된 페르소나 TOP 3와 전체 분포 비율이 시각화되어 나타난다.

---

### Edge Cases

- **데이터 부족 시**: 사용자의 입력 텍스트가 너무 짧아 페르소나 카드를 구성할 키워드가 부족할 경우, 기본 템플릿으로 가이드하거나 추가 입력을 유도합니다.
- **이미지 생성 서버 과부하**: 동시에 많은 사용자가 이미지 생성을 요청할 경우, 큐 시스템이나 클라이언트 사이드 렌더링을 통해 대기 시간을 최소화하고 진행 상태를 표시합니다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-SEC-001**: 시스템은 이미지 생성 및 매칭 요청 시 전달되는 모든 파라미터를 API 경계에서 Zod를 통해 검증해야 한다.
- **FR-SEC-002**: 사용자 간의 궁합 테스트 시, 상대방의 동의(공유 설정)가 있는 데이터에 대해서만 접근을 허용하는 Supabase RLS 정책을 준수해야 한다.
- **FR-001**: 시스템은 사용자의 페르소나 유형에 따른 동적 배경색 및 기하학적 패턴을 포함한 9:16 비율의 이미지를 생성해야 한다.
- **FR-002**: 시스템은 두 페르소나 데이터의 벡터 유사도 또는 키워드 매칭을 통해 100점 만점의 궁합 점수를 계산해야 한다.
- **FR-003**: 시스템은 전체 분석 데이터를 1시간 단위로 집계하여 통계 정보를 Supabase에 캐싱하고 제공해야 한다.
- **FR-004**: 사용자는 자신의 분석 결과 공유 여부를 '공개/비공개'로 설정할 수 있어야 한다.

### Key Entities *(include if feature involves data)*

- **PersonaAnalysis**: 분석 결과, 키워드, 소유자 ID, 공유 설정 상태를 저장합니다.
- **PersonaTrend**: 집계된 페르소나 유형별 빈도수 및 실시간 트렌드 키워드 정보를 포함합니다.
- **MatchingHistory**: (선택 사항) 수행된 궁합 테스트 결과 및 참조된 분석 ID들을 기록합니다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 공유용 카드 이미지 생성 프로세스는 사용자 요청 후 3초 이내에 완료되어야 한다.
- **SC-002**: 분석 완료 사용자 중 최소 25% 이상이 '공유' 또는 '이미지 저장' 버튼을 클릭한다.
- **SC-003**: 신규 유입 사용자의 15% 이상이 페르소나 매칭 링크(궁합 테스트)를 통해 유입된다.
- **SC-004**: 트렌드 대시보드 데이터는 실시간성 유지를 위해 최대 1시간 이내의 오차로 최신화되어야 한다.
